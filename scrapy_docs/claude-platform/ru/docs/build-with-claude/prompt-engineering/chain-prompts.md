# Цепочка сложных подсказок для более сильного результата

Разбивайте сложные задачи на управляемые подзадачи с помощью цепочки подсказок для повышения точности и ясности.

---

<Note>
While these tips apply broadly to all Claude models, you can find prompting tips specific to extended thinking models [here](/docs/en/build-with-claude/prompt-engineering/extended-thinking-tips).
</Note>

При работе со сложными задачами Claude иногда может упустить что-то, если вы попытаетесь решить всё в одной подсказке. Цепочка рассуждений (CoT) — отличный подход, но что если ваша задача состоит из нескольких отдельных этапов, каждый из которых требует глубокого анализа?

Вводим цепочку подсказок: разбиение сложных задач на меньшие, управляемые подзадачи.

## Почему использовать цепочку подсказок?

1. **Точность**: Каждая подзадача получает полное внимание Claude, что снижает ошибки.
2. **Ясность**: Более простые подзадачи означают более чёткие инструкции и результаты.
3. **Отслеживаемость**: Легко определить и исправить проблемы в цепочке подсказок.

***

## Когда использовать цепочку подсказок

Используйте цепочку подсказок для многоэтапных задач, таких как синтез исследований, анализ документов или итеративное создание контента. Когда задача включает несколько преобразований, цитирования или инструкций, цепочка предотвращает упущение или неправильную обработку этапов Claude.

**Помните:** Каждое звено в цепочке получает полное внимание Claude!

<Tip>**Совет по отладке**: Если Claude пропустил этап или работает плохо, изолируйте этот этап в отдельной подсказке. Это позволит вам настроить проблемные этапы без переделки всей задачи.</Tip>

***

## Как создать цепочку подсказок

1. **Определите подзадачи**: Разбейте вашу задачу на отдельные, последовательные этапы.
2. **Структурируйте с помощью XML для чёткой передачи**: Используйте теги XML для передачи результатов между подсказками.
3. **Имейте одну цель для каждой задачи**: Каждая подзадача должна иметь одну чёткую цель.
4. **Итерируйте**: Уточняйте подзадачи на основе производительности Claude.

### Примеры цепочек рабочих процессов:
- **Многоэтапный анализ**: См. примеры с юридическими и бизнес-случаями ниже.
- **Конвейеры создания контента**: Исследование → Структура → Черновик → Редактирование → Форматирование.
- **Обработка данных**: Извлечение → Преобразование → Анализ → Визуализация.
- **Принятие решений**: Сбор информации → Список вариантов → Анализ каждого → Рекомендация.
- **Циклы проверки**: Создание контента → Проверка → Уточнение → Повторная проверка.

<Tip>**Совет по оптимизации**: Для задач с независимыми подзадачами (например, анализ нескольких документов) создайте отдельные подсказки и запустите их параллельно для ускорения.</Tip>

### Продвинутое: Цепочки самокоррекции

Вы можете создать цепочку подсказок, чтобы Claude проверил свою собственную работу! Это выявляет ошибки и уточняет результаты, особенно для критичных задач.

    | Роль | Содержание |

Чтобы повысить до A+, определите аббревиатуры, упомяните исходные статистические данные групп, отметьте любые вторичные результаты и кратко коснитесь необходимости долгосрочных исследований. |
