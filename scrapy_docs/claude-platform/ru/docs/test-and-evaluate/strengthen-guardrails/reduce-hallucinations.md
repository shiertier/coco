# Уменьшение галлюцинаций

---

Даже самые продвинутые языковые модели, такие как Claude, иногда могут генерировать текст, который фактически неверен или не соответствует заданному контексту. Это явление, известное как "галлюцинация", может подорвать надежность ваших решений на базе ИИ.
Это руководство рассмотрит методы минимизации галлюцинаций и обеспечения точности и надежности результатов работы Claude.

## Базовые стратегии минимизации галлюцинаций

- **Позвольте Claude говорить "Я не знаю":** Явно дайте Claude разрешение признавать неуверенность. Эта простая техника может значительно уменьшить количество ложной информации.

<section title="Пример: Анализ отчета о слиянии и поглощении">

| Роль | Содержание |
| ---- | --- |
| Пользователь | Как наш консультант по M&A, проанализируйте этот отчет о потенциальном приобретении AcmeCo компанией ExampleCorp.<br/><br/>\<report><br/>\{\{REPORT}}<br/>\</report><br/><br/>Сосредоточьтесь на финансовых прогнозах, рисках интеграции и нормативных препятствиях. Если вы не уверены в каком-либо аспекте или если в отчете не хватает необходимой информации, скажите "У меня недостаточно информации для уверенной оценки этого." |

</section>

- **Используйте прямые цитаты для фактического обоснования:** Для задач, включающих длинные документы (>20 тыс. токенов), попросите Claude сначала извлечь дословные цитаты перед выполнением задачи. Это основывает его ответы на реальном тексте, уменьшая галлюцинации.

<section title="Пример: Аудит политики конфиденциальности данных">

| Роль | Содержание |
| ---- | --- |
| Пользователь | Как наш специалист по защите данных, проверьте эту обновленную политику конфиденциальности на соответствие GDPR и CCPA.<br/>\<br/>\{\{POLICY}}<br/>\</policy><br/><br/>1. Извлеките точные цитаты из политики, которые наиболее релевантны для соответствия GDPR и CCPA. Если не можете найти релевантные цитаты, укажите "Релевантные цитаты не найдены."<br/><br/>2. Используйте цитаты для анализа соответствия этих разделов политики, ссылаясь на цитаты по номерам. Основывайте свой анализ только на извлеченных цитатах. |

</section>

- **Проверка с помощью цитат**: Сделайте ответ Claude проверяемым, попросив его цитировать источники для каждого утверждения. Вы также можете попросить Claude проверить каждое утверждение, найдя подтверждающую цитату после генерации ответа. Если он не может найти цитату, он должен отозвать утверждение.

<section title="Пример: Составление пресс-релиза о запуске продукта">

| Роль | Содержание |
| ---- | --- |
| Пользователь | Составьте пресс-релиз для нашего нового продукта кибербезопасности, AcmeSecurity Pro, используя только информацию из этих описаний продукта и рыночных отчетов.<br/>\<documents><br/>\{\{DOCUMENTS}}<br/>\</documents><br/><br/>После составления проверьте каждое утверждение в вашем пресс-релизе. Для каждого утверждения найдите прямую цитату из документов, которая его подтверждает. Если вы не можете найти подтверждающую цитату для утверждения, удалите это утверждение из пресс-релиза и отметьте место, где оно было удалено, пустыми [] скобками. |

</section>

## Продвинутые техники

- **Проверка цепочки рассуждений**: Попросите Claude объяснить свои рассуждения пошагово перед предоставлением окончательного ответа. Это может выявить ошибочную логику или предположения.

- **Проверка лучшего из N**: Запустите Claude с одним и тем же промптом несколько раз и сравните результаты. Несоответствия между выводами могут указывать на галлюцинации.

- **Итеративное уточнение**: Используйте выводы Claude как входные данные для последующих промптов, прося его проверить или расширить предыдущие утверждения. Это может помочь обнаружить и исправить несоответствия.

- **Ограничение внешних знаний**: Явно проинструктируйте Claude использовать только информацию из предоставленных документов, а не свои общие знания.

<Note>Помните, что хотя эти техники значительно уменьшают галлюцинации, они не устраняют их полностью. Всегда проверяйте критически важную информацию, особенно для важных решений.</Note>
