# 使用我们的提示词改进器来优化您的提示词

使用我们的提示词改进器来优化您的提示词

---

<Note>
我们的提示词改进器与所有Claude模型兼容，包括那些具有扩展思考能力的模型。有关扩展思考模型的特定提示技巧，请参见[这里](/docs/zh-CN/build-with-claude/extended-thinking)。
</Note>

提示词改进器通过自动化分析和增强帮助您快速迭代和改进提示词。它擅长使提示词对需要高准确性的复杂任务更加稳健。

<Frame>
  ![Image](/docs/images/prompt_improver.png)
</Frame>

## 开始之前

您需要：
- 一个要改进的[提示词模板](/docs/zh-CN/build-with-claude/prompt-engineering/prompt-templates-and-variables)
- 对Claude当前输出问题的反馈（可选但推荐）
- 示例输入和理想输出（可选但推荐）

## 提示词改进器的工作原理

提示词改进器通过4个步骤增强您的提示词：

1. **示例识别**：定位并从您的提示词模板中提取示例
2. **初始草稿**：创建具有清晰部分和XML标签的结构化模板
3. **思维链优化**：添加和完善详细的推理指令
4. **示例增强**：更新示例以演示新的推理过程

您可以在改进模态框中实时观看这些步骤的进行。

## 您将获得什么

提示词改进器生成的模板具有：
- 详细的思维链指令，指导Claude的推理过程并通常改善其性能
- 使用XML标签分离不同组件的清晰组织
- 标准化的示例格式，演示从输入到输出的逐步推理
- 指导Claude初始响应的策略性预填充

<Note>
虽然示例在Workbench UI中单独显示，但它们包含在实际API调用的第一个用户消息的开头。通过点击"**\<\/\> Get Code**"查看原始格式，或通过示例框将示例作为原始文本插入。
</Note>

## 如何使用提示词改进器

1. 提交您的提示词模板
2. 添加关于Claude当前输出问题的任何反馈（例如，"摘要对专家受众来说太基础"）
3. 包含示例输入和理想输出
4. 审查改进后的提示词

## 生成测试示例

还没有示例？使用我们的[测试用例生成器](/docs/zh-CN/test-and-evaluate/eval-tool#creating-test-cases)来：
1. 生成示例输入
2. 获取Claude的响应
3. 编辑响应以匹配您的理想输出
4. 将完善的示例添加到您的提示词中

## 何时使用提示词改进器

提示词改进器最适合：
- 需要详细推理的复杂任务
- 准确性比速度更重要的情况
- Claude当前输出需要显著改进的问题

<Note>
对于延迟或成本敏感的应用程序，考虑使用更简单的提示词。提示词改进器创建的模板会产生更长、更彻底但更慢的响应。
</Note>

## 改进示例

以下是提示词改进器如何增强基本分类提示词的示例：

<section title="原始提示词">

```
从以下Wikipedia文章标题列表中，识别这个句子来自哪篇文章。
只回答文章标题，不要其他内容。

文章标题：
{{titles}}

要分类的句子：
{{sentence}}
```

</section>

<section title="改进后的提示词">

```
您是一个专门将句子与Wikipedia文章标题匹配的智能文本分类系统。您的任务是根据提供的文章标题列表，识别给定句子最可能属于哪篇Wikipedia文章。

首先，审查以下Wikipedia文章标题列表：
<article_titles>
{{titles}}
</article_titles>

现在，考虑这个需要分类的句子：
<sentence_to_classify>
{{sentence}}
</sentence_to_classify>

您的目标是确定提供列表中的哪个文章标题最匹配给定句子。按照以下步骤：

1. 列出句子中的关键概念
2. 将每个关键概念与文章标题进行比较
3. 排列前3个最相关的标题并解释它们为什么相关
4. 选择最能涵盖或与句子内容相关的最合适文章标题

将您的分析包装在<analysis>标签中。包括以下内容：
- 句子中关键概念的列表
- 每个关键概念与文章标题的比较
- 前3个最相关标题的排名及解释
- 您的最终选择和推理

在分析后，提供您的最终答案：列表中最合适的单个Wikipedia文章标题。

只输出选择的文章标题，不要任何额外的文本或解释。
```

</section>

注意改进后的提示词如何：
- 添加清晰的逐步推理指令
- 使用XML标签组织内容
- 提供明确的输出格式要求
- 指导Claude完成分析过程

## 故障排除

常见问题和解决方案：

- **示例未出现在输出中**：检查示例是否使用XML标签正确格式化并出现在第一个用户消息的开头
- **思维链过于冗长**：添加关于期望输出长度和详细程度的具体指令
- **推理步骤不符合您的需求**：修改步骤部分以匹配您的特定用例
