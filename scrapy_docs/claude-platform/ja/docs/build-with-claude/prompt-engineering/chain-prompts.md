# 複雑なプロンプトをチェーンして、より強力なパフォーマンスを実現する

複雑なタスクを小さな管理可能なサブタスクに分割することで、Claude のパフォーマンスを向上させる方法を学びます。

---

<Note>
While these tips apply broadly to all Claude models, you can find prompting tips specific to extended thinking models [here](/docs/en/build-with-claude/prompt-engineering/extended-thinking-tips).
</Note>

複雑なタスクに取り組む場合、単一のプロンプトですべてを処理しようとすると、Claude がうまくいかないことがあります。思考の連鎖（CoT）プロンプティングは素晴らしいですが、それぞれが深い思考を必要とする複数の異なるステップがあるタスクの場合はどうでしょうか？

ここでプロンプトチェーンが登場します。複雑なタスクを小さく、管理可能なサブタスクに分割することです。

## プロンプトをチェーンする理由

1. **精度**: 各サブタスクは Claude の全注意を得るため、エラーが減少します。
2. **明確性**: より単純なサブタスクは、より明確な指示と出力を意味します。
3. **追跡可能性**: プロンプトチェーン内の問題を簡単に特定して修正できます。

***

## プロンプトをチェーンする場合

研究の統合、ドキュメント分析、反復的なコンテンツ作成など、複数ステップのタスクにプロンプトチェーンを使用します。タスクが複数の変換、引用、または指示を含む場合、チェーンは Claude がステップを落とすか誤って処理するのを防ぎます。

**覚えておいてください:** チェーン内の各リンクは Claude の全注意を得ます！

<Tip>**デバッグのヒント**: Claude がステップを見落とすか、パフォーマンスが低い場合は、そのステップを独自のプロンプトで分離します。これにより、タスク全体をやり直すことなく、問題のあるステップを微調整できます。</Tip>

***

## プロンプトをチェーンする方法

1. **サブタスクを特定する**: タスクを異なる順序立ったステップに分割します。
2. **明確なハンドオフのために XML で構造化する**: XML タグを使用して、プロンプト間で出力を渡します。
3. **単一タスクの目標を持つ**: 各サブタスクは単一で明確な目的を持つべきです。
4. **反復する**: Claude のパフォーマンスに基づいてサブタスクを改善します。

### チェーンされたワークフローの例：
- **複数ステップの分析**: 以下の法務および事業例を参照してください。
- **コンテンツ作成パイプライン**: リサーチ → アウトライン → ドラフト → 編集 → フォーマット。
- **データ処理**: 抽出 → 変換 → 分析 → 可視化。
- **意思決定**: 情報収集 → オプションのリスト → 各オプションの分析 → 推奨。
- **検証ループ**: コンテンツ生成 → レビュー → 改善 → 再レビュー。

<Tip>**最適化のヒント**: 独立したサブタスク（複数のドキュメントの分析など）を持つタスクの場合、個別のプロンプトを作成し、速度のために並列で実行します。</Tip>

### 高度な機能: 自己修正チェーン

Claude に自分の仕事をレビューさせるようにプロンプトをチェーンできます！これはエラーをキャッチし、特に高リスクのタスクの出力を改善します。
