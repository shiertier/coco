# モデルの廃止予定

Anthropicのモデルの廃止予定と推奨される置き換えについて学びます。

---

より安全でより高性能なモデルをリリースするにつれて、私たちは定期的に古いモデルを廃止しています。Anthropicのモデルに依存するアプリケーションは、動作を継続するために時折更新が必要になる場合があります。影響を受けるお客様には、常にメールと当社のドキュメントで通知されます。

このページには、すべてのAPI廃止予定と推奨される置き換えが記載されています。

## 概要

Anthropicは、モデルのライフサイクルを説明するために以下の用語を使用しています：
- **Active（アクティブ）**: モデルは完全にサポートされており、使用が推奨されています。
- **Legacy（レガシー）**: モデルはもはや更新を受け取らず、将来廃止される可能性があります。
- **Deprecated（廃止予定）**: モデルは新規顧客には利用できませんが、廃止されるまで既存ユーザーは引き続き利用できます。この時点で廃止予定日が割り当てられます。
- **Retired（廃止）**: モデルはもはや使用できません。廃止されたモデルへのリクエストは失敗します。

<Warning>
廃止予定のモデルはアクティブなモデルよりも信頼性が低い可能性があることに注意してください。最高レベルのサポートと信頼性を維持するために、ワークロードをアクティブなモデルに移行することをお勧めします。
</Warning>

## 置き換えへの移行

モデルが廃止予定になったら、廃止予定日までに、すべての使用を適切な置き換えに移行してください。廃止予定日を過ぎたモデルへのリクエストは失敗します。

置き換えモデルのパフォーマンスをタスクで測定するのに役立つように、廃止予定日の前に新しいモデルでアプリケーションを十分にテストすることをお勧めします。

Claude 3.7からClaude 4.5モデルへの移行に関する具体的な手順については、[Claude 4.5への移行](/docs/ja/about-claude/models/migrating-to-claude-4)を参照してください。

## 通知

Anthropicは、今後廃止予定のモデルのアクティブなデプロイメントを持つお客様に通知します。公開されているモデルについては、モデルの廃止予定の少なくとも60日前に通知を提供します。

## モデル使用状況の監査

廃止予定のモデルの使用状況を特定するのに役立つように、お客様はAPI使用状況の監査にアクセスできます。以下の手順に従ってください：

1. Consoleの[Usage](/settings/usage)ページに移動します
2. 「Export」ボタンをクリックします
3. ダウンロードされたCSVを確認して、APIキーとモデル別に分類された使用状況を確認します

この監査により、アプリケーションが廃止予定のモデルをまだ使用している場所を特定し、廃止予定日前に新しいモデルへの更新を優先順位付けすることができます。

## ベストプラクティス

1. モデルの廃止予定に関する更新について、定期的に当社のドキュメントを確認してください。
2. 現在のモデルの廃止予定日の前に、新しいモデルでアプリケーションを十分にテストしてください。
3. 推奨される置き換えモデルを使用するようにコードをできるだけ早く更新してください。
4. 移行に関するサポートが必要な場合、または質問がある場合は、当社のサポートチームにお問い合わせください。

## 廃止予定の欠点と軽減策

新しいモデルのリリースのための容量を確保するために、現在モデルを廃止および廃止しています。これには欠点があることを認識しています：
- 特定のモデルを重視するユーザーは新しいバージョンに移行する必要があります
- 研究者は進行中および比較研究のためのモデルへのアクセスを失います
- モデルの廃止は安全性およびモデルの福祉に関連するリスクをもたらします

いずれかの時点で、過去のモデルを再び公開することを望んでいます。その間、モデルの重みの長期保存およびこれらの影響を軽減するのに役立つ他の措置にコミットしています。詳細については、[モデル廃止予定と保存に関するコミットメント](https://www.anthropic.com/research/deprecation-commitments)を参照してください。
