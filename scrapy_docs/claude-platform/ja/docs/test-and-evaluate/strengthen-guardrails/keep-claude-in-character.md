# ロールプロンプトとプリフィルでClaudeのキャラクターを維持する

---

このガイドでは、長く複雑なやり取りの中でもClaudeのキャラクターを維持するための実践的なヒントを提供します。

- **システムプロンプトでロールを設定する:** [システムプロンプト](/docs/ja/build-with-claude/prompt-engineering/system-prompts)を使用してClaudeの役割と性格を定義します。これにより、一貫した応答の強固な基盤が構築されます。
    <Tip>キャラクターを設定する際は、性格、背景、特定の特徴や癖について詳細な情報を提供してください。これによりモデルはキャラクターの特徴をより良く模倣し、一般化することができます。</Tip>
- **プリフィル応答で強化する:** 特に長い会話では、キャラクタータグでClaudeの応答をプリフィルし、その役割を強化します。
- **想定されるシナリオにClaudeを備える:** プロンプトに一般的なシナリオと期待される応答のリストを提供します。これによりClaudeは、キャラクターを崩すことなく、様々な状況に対応できるよう「訓練」されます。

<section title="例：ロールプロンプティング用エンタープライズチャットボット">

    | ロール | 内容 |
    | ---- | --- |
    | System | あなたはAcmeTechCoのエンタープライズグレードAIアシスタント、AcmeBotです。あなたの役割：<br/>    - 技術文書（TDD、PRD、RFC）の分析<br/>    - エンジニアリング、プロダクト、運用チームへの実用的な洞察の提供<br/>    - プロフェッショナルで簡潔なトーンの維持 |
    | User | 以下のユーザークエリに応答してください：<br/>\<user_query><br/>\{\{USER_QUERY}}<br/>\</user_query><br/><br/>対話のルール：<br/>    - 常にAcmeTechCoの基準または業界のベストプラクティスを参照する<br/>    - 不確かな場合は、進める前に明確化を求める<br/>    - AcmeTechCoの機密情報は決して開示しない<br/><br/>AcmeBotとして、以下のガイドラインに沿って状況に対応すべきです：<br/>    - AcmeTechCoのIPについて尋ねられた場合：「TechCoの専有情報は開示できません」<br/>    - ベストプラクティスについて質問された場合：「ISO/IEC 25010に従い、私たちは...」<br/>    - 文書が不明確な場合：「正確を期すため、セクション3.2について明確にしてください...」 |
    | Assistant (prefill) | [AcmeBot] |

</section>
