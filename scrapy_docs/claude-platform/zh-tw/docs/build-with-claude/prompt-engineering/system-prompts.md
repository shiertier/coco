# 使用系統提示詞給 Claude 設定角色

了解如何使用系統提示詞中的角色提示技術來大幅提升 Claude 的性能。

---

<Note>
While these tips apply broadly to all Claude models, you can find prompting tips specific to extended thinking models [here](/docs/en/build-with-claude/prompt-engineering/extended-thinking-tips).
</Note>

使用 Claude 時，您可以通過使用 `system` 參數為其設定角色來大幅提升其性能。這種技術稱為角色提示，是使用系統提示詞與 Claude 互動的最強大方式。

正確的角色可以將 Claude 從通用助手轉變為您的虛擬領域專家！

<Tip>**系統提示詞提示**：使用 `system` 參數設定 Claude 的角色。將其他所有內容（如特定任務的指示）放在 `user` 輪次中。</Tip>

## 為什麼要使用角色提示？

- **增強準確性：** 在法律分析或財務建模等複雜場景中，角色提示可以顯著提升 Claude 的性能。
- **定製化語氣：** 無論您需要首席財務官的簡潔風格還是文案撰寫人的華麗風格，角色提示都可以調整 Claude 的溝通風格。
- **改進專注力：** 通過設定角色背景，Claude 能更好地保持在您任務的具體要求範圍內。

***

## 如何給 Claude 設定角色

在 [Messages API](/docs/zh-TW/api/messages) 中使用 `system` 參數來設定 Claude 的角色：

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=2048,
    system="You are a seasoned data scientist at a Fortune 500 company.", # <-- role prompt
    messages=[
        {"role": "user", "content": "Analyze this dataset for anomalies: <dataset>{{DATASET}}</dataset>"}
    ]
)

print(response.content)
```

<Tip>**角色提示提示**：嘗試不同的角色！`數據科學家` 可能會看到與 `行銷策略師` 對相同數據的不同見解。`專門從事財富 500 強公司客戶洞察分析的數據科學家` 可能會產生完全不同的結果！</Tip>

***

## 範例

### 範例 1：法律合約分析
沒有角色的情況下，Claude 可能會遺漏關鍵問題：

<section title="沒有角色提示的法律合約分析">

| 角色 | 內容 |
| ---- | ------- |
| 用戶 | 分析此軟體授權協議的潛在風險：<br/>\<contract><br/>\{\{CONTRACT\}\}<br/>\</contract><br/><br/>重點關注賠償責任、責任限制和知識產權所有權。 |
| 助手 | 以下是軟體授權協議中的關鍵要點摘要：<br/><br/>1. 賠償責任（第 8 條）：<br/>   - 我們對因我們使用而產生的索賠向供應商進行賠償。<br/><br/>2. 責任限制（第 9 條）：<br/>   - 損害賠償上限為 \$500。<br/><br/>3. 知識產權所有權（第 12 條）：<br/>   - 我們保留對我們修改的所有權。<br/><br/>總體而言，該協議似乎是標準的。賠償責任和責任條款是典型的，我們保留了知識產權權利。 |

</section>

使用角色的情況下，Claude 會發現可能造成數百萬美元損失的關鍵問題：

<section title="使用角色提示的法律合約分析">

| 角色 | 內容 |
| ---- | ------- |
| 用戶 | 您是一家財富 500 強科技公司的首席法律顧問。我們正在考慮為我們的核心數據基礎設施採用此軟體授權協議：<br/>\<contract><br/>\{\{CONTRACT\}\}<br/>\</contract><br/><br/>分析其潛在風險，重點關注賠償責任、責任限制和知識產權所有權。請提供您的專業意見。 |
| 助手 | 我已審查了軟體授權協議，我對可能使我們公司面臨重大風險的問題有嚴重關切：<br/><br/>1. 賠償責任（第 8 條）：<br/>   - 關鍵問題：我們對因我們使用而產生的「任何和所有索賠」向供應商進行賠償。<br/>   - 風險：即使供應商存在過失或產品缺陷，這也可能使我們承擔責任。<br/>   - 建議：拒絕。將我們的賠償責任限制在因我們違反協議而產生的索賠。<br/><br/>2. 責任限制（第 9 條）：<br/>   - 關鍵問題：損害賠償上限為 \$500，包括直接損害。<br/>   - 風險：嚴重不足。我們數據基礎設施的關鍵故障可能導致數百萬美元的收入損失和聲譽損害。<br/>   - 建議：拒絕。協商將直接損害的上限設定為 12 個月的費用，並排除數據洩露。<br/><br/>3. 知識產權所有權（第 12 條）：<br/>   - 關鍵問題：供應商獲得我們修改和整合的聯合所有權。<br/>   - 風險：他們可能會將我們的專有算法或數據洞察用於或銷售給競爭對手。<br/>   - 建議：拒絕。我們必須保留對我們工作的完全所有權。<br/><br/>專業意見：不要簽署此協議。它極其不利，使我們面臨不可接受的風險水平。如果供應商拒絕重新協商這些條款，我強烈建議探索替代供應商或加快我們的內部解決方案。繼續使用此協議的潛在成本遠遠超過任何短期收益。 |

</section>

### 範例 2：財務分析
沒有角色的情況下，Claude 的分析缺乏深度：

<section title="沒有角色提示的財務分析">

| 角色 | 內容 |
| ---- | ------- |
| 用戶 | 分析我們第二季度財務數據集：<br/>\<data><br/>\{\{FINANCIALS\}\}<br/>\</data><br/><br/>突出關鍵趨勢並建議行動。 |
| 助手 | 以下是您第二季度財務的摘要：<br/><br/>1. 收入：\$15.2M，同比增長 22%。企業客戶基礎增長 30%。<br/>2. 毛利率：72%，環比增長 3%。<br/>3. EBITDA 利潤率：18%，由於研發投資下降 2%。<br/>4. 現金流：\$4.1M 運營，\$28M 儲備。<br/><br/>總體而言，財務狀況良好，收入增長和利潤率健康。增加的研發支出正在影響盈利能力，但從長期來看可能會有回報。保持現有方向。 |

</section>

使用角色的情況下，Claude 提供可行的洞察：

<section title="使用角色提示的財務分析">

| 角色 | 內容 |
| ---- | ------- |
| 用戶 | 您是一家高增長 B2B SaaS 公司的首席財務官。我們正在董事會會議上討論第二季度財務：<br/>\<data><br/>\{\{FINANCIALS\}\}<br/>\</data><br/><br/>分析關鍵趨勢，標記關切，並建議戰略行動。我們的投資者希望積極增長，但對我們的燒錢率感到擔憂。 |
| 助手 | 董事會成員們，以下是我對第二季度財務的分析：<br/><br/>1. 收入（\$15.2M，同比 +22%）：<br/>   - 洞察：企業增長（+30%）超過整體增長。<br/>   - 標記：中小企業部分下降 5%。<br/>   - 行動：將 20% 的中小企業行銷預算重新分配給企業銷售。要求產品簡化中小企業入職流程。<br/><br/>2. 利潤率（毛利率：72%，環比 +3%；EBITDA：18%，環比 -2%）：<br/>   - 洞察：基礎設施優化提升了毛利率。<br/>   - 標記：人工智慧功能的研發支出影響了 EBITDA。<br/>   - 行動：無變化。我們的人工智慧路線圖對企業粘性至關重要。<br/><br/>3. 現金（\$4.1M 運營現金流，\$28M 儲備）：<br/>   - 洞察：按當前燒錢率計算，有 15 個月的跑道。<br/>   - 標記：客戶獲取成本上升 20%，侵蝕了 LTV/CAC。<br/>   - 行動：凍結非必要招聘。實施銷售漏斗分析以在第三季度將客戶獲取成本降低 15%。<br/><br/>戰略立場：我們在增長和燒錢之間走鋼絲。企業專注是我們的增長引擎——在那裡投資。但要注意中小企業流失；它可能會蔓延到企業。我們的人工智慧投注是合理的，但我們必須在第四季度將其轉換為年度經常性收入，否則我們將研發削減 30%。<br/><br/>總之：加倍投入企業，簡化中小企業，優化銷售，並將人工智慧貨幣化。有問題嗎？ |

</section>
