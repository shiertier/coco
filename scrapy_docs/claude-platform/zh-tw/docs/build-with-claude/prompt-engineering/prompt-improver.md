# 使用我們的提示詞改進器來優化您的提示詞

使用我們的提示詞改進器來優化您的提示詞

---

<Note>
我們的提示詞改進器與所有 Claude 模型相容，包括具有擴展思考能力的模型。有關擴展思考模型的特定提示詞技巧，請參閱[這裡](/docs/zh-TW/build-with-claude/extended-thinking)。
</Note>

提示詞改進器透過自動化分析和增強功能，幫助您快速迭代和改進提示詞。它擅長讓提示詞對需要高準確性的複雜任務更加穩健。

<Frame>
  ![Image](/docs/images/prompt_improver.png)
</Frame>

## 開始之前

您需要：
- 一個要改進的[提示詞模板](/docs/zh-TW/build-with-claude/prompt-engineering/prompt-templates-and-variables)
- 對 Claude 當前輸出問題的反饋（可選但建議提供）
- 範例輸入和理想輸出（可選但建議提供）

## 提示詞改進器的工作原理

提示詞改進器透過 4 個步驟來增強您的提示詞：

1. **範例識別**：定位並從您的提示詞模板中提取範例
2. **初始草稿**：創建具有清晰部分和 XML 標籤的結構化模板
3. **思維鏈精煉**：添加並精煉詳細的推理指令
4. **範例增強**：更新範例以展示新的推理過程

您可以在改進模態中即時觀看這些步驟的進行。

## 您將獲得什麼

提示詞改進器生成的模板具有：
- 詳細的思維鏈指令，引導 Claude 的推理過程並通常改善其性能
- 使用 XML 標籤分離不同組件的清晰組織
- 標準化的範例格式，展示從輸入到輸出的逐步推理
- 引導 Claude 初始回應的策略性預填充

<Note>
雖然範例在 Workbench UI 中單獨顯示，但它們包含在實際 API 調用的第一個用戶消息的開頭。通過點擊「**\<\/\> Get Code**」查看原始格式，或通過範例框將範例作為原始文本插入。
</Note>

## 如何使用提示詞改進器

1. 提交您的提示詞模板
2. 添加關於 Claude 當前輸出問題的任何反饋（例如，「摘要對專家受眾來說太基礎了」）
3. 包含範例輸入和理想輸出
4. 審查改進後的提示詞

## 生成測試範例

還沒有範例？使用我們的[測試案例生成器](/docs/zh-TW/test-and-evaluate/eval-tool#creating-test-cases)來：
1. 生成範例輸入
2. 獲得 Claude 的回應
3. 編輯回應以匹配您的理想輸出
4. 將完善的範例添加到您的提示詞中

## 何時使用提示詞改進器

提示詞改進器最適合：
- 需要詳細推理的複雜任務
- 準確性比速度更重要的情況
- Claude 當前輸出需要顯著改進的問題

<Note>
對於延遲或成本敏感的應用程序，請考慮使用更簡單的提示詞。提示詞改進器創建的模板會產生更長、更徹底但更慢的回應。
</Note>

## 改進範例

以下是提示詞改進器如何增強基本分類提示詞：

<section title="原始提示詞">

```
從以下 Wikipedia 文章標題列表中，識別這個句子來自哪篇文章。
只回應文章標題，不要其他內容。

文章標題：
{{titles}}

要分類的句子：
{{sentence}}
```

</section>

<section title="改進後的提示詞">

```
您是一個專門將句子與 Wikipedia 文章標題匹配的智能文本分類系統。您的任務是根據提供的文章標題列表，識別給定句子最可能屬於哪篇 Wikipedia 文章。

首先，審查以下 Wikipedia 文章標題列表：
<article_titles>
{{titles}}
</article_titles>

現在，考慮這個需要分類的句子：
<sentence_to_classify>
{{sentence}}
</sentence_to_classify>

您的目標是確定提供列表中的哪個文章標題最符合給定句子。請遵循以下步驟：

1. 列出句子中的關鍵概念
2. 將每個關鍵概念與文章標題進行比較
3. 排列前 3 個最相關的標題並解釋它們為什麼相關
4. 選擇最能涵蓋或與句子內容相關的最合適文章標題

將您的分析包裝在 <analysis> 標籤中。包括以下內容：
- 句子中關鍵概念的列表
- 每個關鍵概念與文章標題的比較
- 前 3 個最相關標題的排名及解釋
- 您的最終選擇和推理

在分析後，提供您的最終答案：列表中最合適的單一 Wikipedia 文章標題。

僅輸出選擇的文章標題，不要任何額外的文本或解釋。
```

</section>

注意改進後的提示詞如何：
- 添加清晰的逐步推理指令
- 使用 XML 標籤組織內容
- 提供明確的輸出格式要求
- 引導 Claude 完成分析過程

## 故障排除

常見問題和解決方案：

- **範例未出現在輸出中**：檢查範例是否使用 XML 標籤正確格式化，並出現在第一個用戶消息的開頭
- **思維鏈過於冗長**：添加關於所需輸出長度和詳細程度的具體指令
- **推理步驟不符合您的需求**：修改步驟部分以匹配您的特定用例
