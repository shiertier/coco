# Версии

При выполнении API-запросов вы должны отправлять заголовок запроса `anthropic-version`. Например, `anthropic-version: 2023-06-01`. Если вы используете наши [клиентские SDK](/docs/ru/api/client-sdks), это обрабатывается автоматически.

---

Для любой данной версии API мы сохраним:

* Существующие входные параметры
* Существующие выходные параметры

Однако мы можем сделать следующее:

* Добавить дополнительные необязательные входные данные
* Добавить дополнительные значения к выходным данным
* Изменить условия для определенных типов ошибок
* Добавить новые варианты к значениям вывода типа enum (например, типы событий потоковой передачи)

В общем, если вы используете API, как описано в этом справочнике, мы не нарушим ваше использование.

## История версий

Мы всегда рекомендуем использовать последнюю версию API, когда это возможно. Предыдущие версии считаются устаревшими и могут быть недоступны для новых пользователей.

* `2023-06-01`  
   * Новый формат для [потоковых](/docs/ru/api/streaming) событий, отправляемых сервером (SSE):  
         * Завершения являются инкрементальными. Например, `" Hello"`, `" my"`, `" name"`, `" is"`, `" Claude." ` вместо `" Hello"`, `" Hello my"`, `" Hello my name"`, `" Hello my name is"`, `" Hello my name is Claude."`.  
         * Все события являются [именованными событиями](https://developer.mozilla.org/en-US/Web/API/Server-sent%5Fevents/Using%5Fserver-sent%5Fevents#named%5Fevents), а не [событиями только с данными](https://developer.mozilla.org/en-US/Web/API/Server-sent%5Fevents/Using%5Fserver-sent%5Fevents#data-only%5Fmessages).  
         * Удалено ненужное событие `data: [DONE]`.  
   * Удалены устаревшие значения `exception` и `truncated` в ответах.
* `2023-01-01`: Первоначальный выпуск.
