# Снятие моделей с поддержки

Информация о снятии моделей Claude с поддержки, рекомендуемых заменах и сроках выхода из обслуживания.

---

По мере выпуска более безопасных и мощных моделей мы регулярно снимаем с поддержки старые модели. Приложения, использующие модели Anthropic, могут потребовать периодических обновлений для продолжения работы. Затронутые клиенты всегда будут уведомлены по электронной почте и в нашей документации.

На этой странице перечислены все снятия API с поддержки вместе с рекомендуемыми заменами.

## Обзор

Anthropic использует следующие термины для описания жизненного цикла наших моделей:
- **Active**: Модель полностью поддерживается и рекомендуется для использования.
- **Legacy**: Модель больше не будет получать обновления и может быть снята с поддержки в будущем.
- **Deprecated**: Модель больше не доступна для новых клиентов, но продолжает быть доступной для существующих пользователей до снятия с обслуживания. На этом этапе мы назначаем дату выхода из обслуживания.
- **Retired**: Модель больше не доступна для использования. Запросы к снятым с обслуживания моделям будут отклонены.

<Warning>
Обратите внимание, что снятые с поддержки модели, вероятно, будут менее надежными, чем активные модели. Мы настоятельно рекомендуем вам перенести рабочие нагрузки на активные модели, чтобы поддерживать наивысший уровень поддержки и надежности.
</Warning>

## Миграция на замены

После снятия модели с поддержки, пожалуйста, перенесите все использование на подходящую замену до даты выхода из обслуживания. Запросы к моделям после даты выхода из обслуживания будут отклонены.

Чтобы помочь измерить производительность моделей-замен на ваших задачах, мы рекомендуем тщательное тестирование ваших приложений с новыми моделями задолго до даты выхода из обслуживания.

Для получения конкретных инструкций по миграции с Claude 3.7 на модели Claude 4.5 см. [Миграция на Claude 4.5](/docs/ru/about-claude/models/migrating-to-claude-4).

## Уведомления

Anthropic уведомляет клиентов с активными развертываниями о моделях с предстоящим снятием с обслуживания. Мы предоставляем как минимум 60 дней уведомления перед снятием модели с обслуживания для публично выпущенных моделей.

## Аудит использования модели

Чтобы помочь определить использование снятых с поддержки моделей, клиенты могут получить доступ к аудиту использования их API. Выполните следующие шаги:

1. Перейдите на страницу [Usage](/settings/usage) в Console
2. Нажмите кнопку "Export"
3. Просмотрите загруженный CSV-файл, чтобы увидеть использование, разбитое по ключам API и моделям

Этот аудит поможет вам найти любые экземпляры, где ваше приложение все еще использует снятые с поддержки модели, позволяя вам приоритизировать обновления на более новые модели до даты выхода из обслуживания.

## Лучшие практики

1. Регулярно проверяйте нашу документацию на предмет обновлений о снятии моделей с поддержки.
2. Тестируйте ваши приложения с более новыми моделями задолго до даты выхода из обслуживания вашей текущей модели.
3. Обновите ваш код для использования рекомендуемой модели-замены как можно скорее.
4. Свяжитесь с нашей командой поддержки, если вам нужна помощь с миграцией или у вас есть какие-либо вопросы.

## Недостатки снятия с поддержки и их смягчение

Мы в настоящее время снимаем с поддержки и выводим из обслуживания модели, чтобы обеспечить емкость для новых выпусков моделей. Мы признаем, что это имеет недостатки:
- Пользователи, которые ценят конкретные модели, должны перейти на новые версии
- Исследователи теряют доступ к моделям для текущих и сравнительных исследований
- Снятие модели с обслуживания вводит риски, связанные с безопасностью и благополучием модели

В какой-то момент мы надеемся снова сделать прошлые модели общедоступными. Тем временем мы взяли на себя обязательство по долгосрочному сохранению весов моделей и другим мерам для помощи в смягчении этих последствий. Для получения дополнительной информации см. [Commitments on Model Deprecation and Preservation](https://www.anthropic.com/research/deprecation-commitments).
