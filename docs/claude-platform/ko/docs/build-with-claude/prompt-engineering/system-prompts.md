# 시스템 프롬프트로 Claude에게 역할 부여하기

Claude의 성능을 극대화하기 위해 시스템 프롬프트를 사용하여 역할을 부여하는 방법을 알아봅니다.

---

<Note>
While these tips apply broadly to all Claude models, you can find prompting tips specific to extended thinking models [here](/docs/en/build-with-claude/prompt-engineering/extended-thinking-tips).
</Note>

Claude를 사용할 때, `system` 매개변수를 사용하여 역할을 부여함으로써 성능을 극적으로 향상시킬 수 있습니다. 이 기법은 역할 프롬프팅(role prompting)으로 알려져 있으며, Claude와 함께 시스템 프롬프트를 사용하는 가장 강력한 방법입니다.

올바른 역할은 Claude를 일반 어시스턴트에서 당신의 가상 도메인 전문가로 변환할 수 있습니다!

<Tip>**시스템 프롬프트 팁**: `system` 매개변수를 사용하여 Claude의 역할을 설정하세요. 작업별 지침과 같은 다른 모든 것은 `user` 턴에 넣으세요.</Tip>

## 역할 프롬프팅을 사용하는 이유는 무엇인가요?

- **향상된 정확성:** 법률 분석이나 재무 모델링과 같은 복잡한 시나리오에서 역할 프롬프팅은 Claude의 성능을 크게 향상시킬 수 있습니다.
- **맞춤형 톤:** CFO의 간결함이 필요하든 카피라이터의 재치가 필요하든, 역할 프롬프팅은 Claude의 커뮤니케이션 스타일을 조정합니다.
- **개선된 집중력:** 역할 컨텍스트를 설정함으로써 Claude는 작업의 특정 요구사항 범위 내에서 더 잘 유지됩니다.

***

## Claude에게 역할을 부여하는 방법

[Messages API](/docs/ko/api/messages)에서 `system` 매개변수를 사용하여 Claude의 역할을 설정하세요:

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=2048,
    system="You are a seasoned data scientist at a Fortune 500 company.", # <-- role prompt
    messages=[
        {"role": "user", "content": "Analyze this dataset for anomalies: <dataset>{{DATASET}}</dataset>"}
    ]
)

print(response.content)
```

<Tip>**역할 프롬프팅 팁**: 역할을 실험해보세요! `데이터 과학자`는 같은 데이터에 대해 `마케팅 전략가`와 다른 인사이트를 볼 수 있습니다. `Fortune 500 기업의 고객 인사이트 분석을 전문으로 하는 데이터 과학자`는 여전히 다른 결과를 낼 수 있습니다!</Tip>

***

## 예제

### 예제 1: 법률 계약 분석
역할 없이 Claude는 중요한 문제를 놓칠 수 있습니다:

<section title="역할 프롬프팅 없는 법률 계약 분석">

| 역할 | 내용 |
| ---- | ------- |
| 사용자 | 이 소프트웨어 라이선싱 계약을 잠재적 위험에 대해 분석하세요:<br/>\<contract><br/>\{\{CONTRACT\}\}<br/>\</contract><br/><br/>면책, 책임, IP 소유권에 초점을 맞추세요. |
| 어시스턴트 | 소프트웨어 라이선싱 계약의 주요 포인트를 요약하겠습니다:<br/><br/>1. 면책(섹션 8):<br/>   - 우리는 우리의 사용으로 인한 청구에 대해 공급업체를 면책합니다.<br/><br/>2. 책임 제한(섹션 9):<br/>   - 손해배상액은 $500으로 제한됩니다.<br/><br/>3. IP 소유권(섹션 12):<br/>   - 우리는 우리의 수정사항에 대한 소유권을 유지합니다.<br/><br/>전반적으로 계약은 표준적입니다. 면책 및 책임 조항은 일반적이며, 우리는 IP 권리를 유지합니다. |

</section>

역할을 사용하면 Claude는 수백만 달러의 손실을 초래할 수 있는 중요한 문제를 포착합니다:

<section title="역할 프롬프팅을 사용한 법률 계약 분석">

| 역할 | 내용 |
| ---- | ------- |
| 사용자 | 당신은 Fortune 500 기술 회사의 법무 담당 임원입니다. 우리는 핵심 데이터 인프라를 위해 이 소프트웨어 라이선싱 계약을 고려하고 있습니다:<br/>\<contract><br/>\{\{CONTRACT\}\}<br/>\</contract><br/><br/>면책, 책임, IP 소유권에 초점을 맞춰 잠재적 위험을 분석하세요. 전문적인 의견을 제시하세요. |
| 어시스턴트 | 소프트웨어 라이선싱 계약을 검토했으며, 우리 회사를 상당한 위험에 노출시킬 수 있는 심각한 우려 사항이 있습니다:<br/><br/>1. 면책(섹션 8):<br/>   - 중요 문제: 우리는 우리의 사용으로 인한 "모든 청구"에 대해 공급업체를 면책합니다.<br/>   - 위험: 공급업체의 과실이나 제품 결함에 대해서도 우리에게 책임이 있을 수 있습니다.<br/>   - 권장사항: 거절. 우리의 면책을 계약 위반으로 인한 청구로 제한하세요.<br/><br/>2. 책임 제한(섹션 9):<br/>   - 중요 문제: 직접 손해배상을 포함한 손해배상액이 $500으로 제한됩니다.<br/>   - 위험: 극도로 부족합니다. 데이터 인프라의 중요한 장애는 수백만 달러의 수익 손실과 평판 손상을 초래할 수 있습니다.<br/>   - 권장사항: 거절. 직접 손해배상의 경우 월 수수료의 12개월로 상한을 협상하고, 데이터 침해에 대한 예외를 포함하세요.<br/><br/>3. IP 소유권(섹션 12):<br/>   - 중요 문제: 공급업체는 우리의 수정사항 및 통합에 대한 공동 소유권을 얻습니다.<br/>   - 위험: 그들은 우리의 독점 알고리즘이나 데이터 인사이트를 경쟁사에 사용하거나 판매할 수 있습니다.<br/>   - 권장사항: 거절. 우리는 우리의 작업에 대한 완전한 소유권을 유지해야 합니다.<br/><br/>전문적 의견: 이 계약에 서명하지 마세요. 극도로 불리하며 수용할 수 없는 수준의 위험에 우리를 노출시킵니다. 공급업체가 이러한 조건을 재협상하기를 거부하면, 대체 공급업체를 탐색하거나 사내 솔루션 개발을 가속화할 것을 강력히 권고합니다. 이 계약을 진행하는 잠재적 비용은 단기 이득을 훨씬 초과합니다. |

</section>

### 예제 2: 재무 분석
역할 없이 Claude의 분석은 깊이가 부족합니다:

<section title="역할 프롬프팅 없는 재무 분석">

| 역할 | 내용 |
| ---- | ------- |
| 사용자 | 우리의 Q2 재무 데이터를 분석하세요:<br/>\<data><br/>\{\{FINANCIALS\}\}<br/>\</data><br/><br/>주요 추세를 강조하고 조치를 권장하세요. |
| 어시스턴트 | 다음은 Q2 재무의 요약입니다:<br/><br/>1. 수익: $15.2M, 전년 대비 22% 증가. 엔터프라이즈 클라이언트 기반 30% 증가.<br/>2. 총이익률: 72%, Q1 대비 3% 증가.<br/>3. EBITDA 마진: 18%, R&D 투자로 인해 Q1 대비 2% 감소.<br/>4. 현금 흐름: $4.1M 운영, $28M 준비금.<br/><br/>전반적으로 수익 성장과 건강한 마진으로 재무가 강합니다. R&D 지출 증가는 수익성에 영향을 미치지만 장기적으로 보상될 수 있습니다. 현재 방향을 유지하세요. |

</section>

역할을 사용하면 Claude는 실행 가능한 인사이트를 제공합니다:

<section title="역할 프롬프팅을 사용한 재무 분석">

| 역할 | 내용 |
| ---- | ------- |
| 사용자 | 당신은 고성장 B2B SaaS 회사의 CFO입니다. 우리는 Q2 재무를 논의하는 이사회 회의에 있습니다:<br/>\<data><br/>\{\{FINANCIALS\}\}<br/>\</data><br/><br/>주요 추세를 분석하고, 우려 사항을 지적하며, 전략적 조치를 권장하세요. 우리의 투자자들은 공격적인 성장을 원하지만 우리의 소진율에 대해 우려하고 있습니다. |
| 어시스턴트 | 이사회 위원들, 다음은 Q2 재무 분석입니다:<br/><br/>1. 수익($15.2M, +22% YoY):<br/>   - 인사이트: 엔터프라이즈 성장(+30%)이 전체 성장을 능가하고 있습니다.<br/>   - 경고: SMB 세그먼트 5% 감소.<br/>   - 조치: SMB 마케팅 예산의 20%를 엔터프라이즈 판매로 재배치하세요. 제품팀이 SMB 온보딩을 단순화하도록 하세요.<br/><br/>2. 마진(총이익: 72%, +3% QoQ; EBITDA: 18%, -2% QoQ):<br/>   - 인사이트: 인프라 최적화가 총이익률을 향상시켰습니다.<br/>   - 경고: AI 기능에 대한 R&D 지출이 EBITDA를 감소시켰습니다.<br/>   - 조치: 변경 없음. 우리의 AI 로드맵은 엔터프라이즈 고착성에 중요합니다.<br/><br/>3. 현금($4.1M OCF, $28M 준비금):<br/>   - 인사이트: 현재 소진율로 15개월 활주로.<br/>   - 경고: CAC 20% 증가, LTV/CAC 악화.<br/>   - 조치: 필수 불가결하지 않은 채용을 동결하세요. Q3에서 CAC를 15% 줄이기 위해 판매 펀넬 분석을 구현하세요.<br/><br/>전략적 입장: 우리는 성장과 소진 사이의 균형을 맞추고 있습니다. 엔터프라이즈 초점이 우리의 성장 엔진입니다—거기에 투자하세요. 하지만 SMB 이탈을 주시하세요; 엔터프라이즈로 확산될 수 있습니다. 우리의 AI 베팅은 타당하지만, Q4까지 ARR로 전환해야 하거나, R&D를 30% 줄여야 합니다.<br/><br/>요약하면: 엔터프라이즈에 집중하고, SMB를 간소화하고, 판매를 최적화하고, AI를 수익화하세요. 질문이 있으신가요? |

</section>
