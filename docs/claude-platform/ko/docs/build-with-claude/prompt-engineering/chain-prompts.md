# 더 강력한 성능을 위해 복잡한 프롬프트 연결하기

복잡한 작업을 더 작은 관리 가능한 부분 작업으로 나누어 Claude의 성능을 향상시키는 프롬프트 체이닝 기법을 배웁니다.

---

<Note>
While these tips apply broadly to all Claude models, you can find prompting tips specific to extended thinking models [here](/docs/en/build-with-claude/prompt-engineering/extended-thinking-tips).
</Note>

복잡한 작업을 다룰 때, 모든 것을 단일 프롬프트에서 처리하려고 하면 Claude가 실수할 수 있습니다. 사고의 연쇄(CoT) 프롬프팅은 훌륭하지만, 각각 깊이 있는 사고가 필요한 여러 개의 서로 다른 단계가 있는 작업이라면 어떻게 할까요?

프롬프트 체이닝이 등장합니다: 복잡한 작업을 더 작고 관리 가능한 부분 작업으로 나누는 것입니다.

## 프롬프트를 연결하는 이유는 무엇인가요?

1. **정확성**: 각 부분 작업이 Claude의 완전한 주의를 받아 오류를 줄입니다.
2. **명확성**: 더 간단한 부분 작업은 더 명확한 지시사항과 결과를 의미합니다.
3. **추적 가능성**: 프롬프트 체인에서 문제를 쉽게 찾아내고 수정할 수 있습니다.

***

## 프롬프트를 연결할 때

연구 종합, 문서 분석 또는 반복적인 콘텐츠 생성과 같은 다단계 작업에 프롬프트 체이닝을 사용하세요. 작업에 여러 변환, 인용 또는 지시사항이 포함될 때, 체이닝은 Claude가 단계를 놓치거나 잘못 처리하는 것을 방지합니다.

**기억하세요:** 체인의 각 링크가 Claude의 완전한 주의를 받습니다!

<Tip>**디버깅 팁**: Claude가 단계를 놓치거나 성능이 좋지 않으면, 해당 단계를 자체 프롬프트에서 분리하세요. 이렇게 하면 전체 작업을 다시 하지 않고도 문제가 있는 단계를 미세 조정할 수 있습니다.</Tip>

***

## 프롬프트를 연결하는 방법

1. **부분 작업 식별**: 작업을 서로 다른 순차적 단계로 나누세요.
2. **명확한 인수를 위해 XML로 구조화**: XML 태그를 사용하여 프롬프트 간에 출력을 전달하세요.
3. **단일 작업 목표 설정**: 각 부분 작업은 단일하고 명확한 목표를 가져야 합니다.
4. **반복**: Claude의 성능을 기반으로 부분 작업을 개선하세요.

### 예시 연결된 워크플로우:
- **다단계 분석**: 아래의 법률 및 비즈니스 예시를 참조하세요.
- **콘텐츠 생성 파이프라인**: 연구 → 개요 → 초안 → 편집 → 형식화.
- **데이터 처리**: 추출 → 변환 → 분석 → 시각화.
- **의사결정**: 정보 수집 → 옵션 나열 → 각각 분석 → 권장.
- **검증 루프**: 콘텐츠 생성 → 검토 → 개선 → 재검토.

<Tip>**최적화 팁**: 독립적인 부분 작업이 있는 작업(예: 여러 문서 분석)의 경우, 별도의 프롬프트를 만들고 속도를 위해 병렬로 실행하세요.</Tip>
