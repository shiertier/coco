# 올바른 모델 선택하기

Claude 모델을 선택할 때는 세 가지 핵심 요소인 기능, 속도, 비용의 균형을 맞춰야 합니다. 이 가이드는 특정 요구사항에 따라 정보에 입각한 결정을 내릴 수 있도록 도와줍니다.

---

## 핵심 기준 수립

Claude 모델을 선택할 때는 먼저 다음 요소들을 평가하는 것을 권장합니다:
- **기능:** 요구사항을 충족하기 위해 모델이 가져야 할 특정 기능이나 능력은 무엇입니까?
- **속도:** 애플리케이션에서 모델이 얼마나 빠르게 응답해야 합니까?
- **비용:** 개발 및 프로덕션 사용에 대한 예산은 얼마입니까?

이러한 답변을 미리 알고 있으면 사용할 모델을 좁혀서 결정하기가 훨씬 쉬워집니다.

***

## 시작하기에 가장 좋은 모델 선택

어떤 Claude 모델이 요구사항에 가장 잘 맞는지 테스트하기 위해 사용할 수 있는 두 가지 일반적인 접근 방식이 있습니다.

### 옵션 1: 빠르고 비용 효율적인 모델로 시작

많은 애플리케이션의 경우, Claude Haiku 4.5와 같은 더 빠르고 비용 효율적인 모델로 시작하는 것이 최적의 접근 방식이 될 수 있습니다:

1. Claude Haiku 4.5로 구현 시작
2. 사용 사례를 철저히 테스트
3. 성능이 요구사항을 충족하는지 평가
4. 특정 기능 격차가 있을 경우에만 업그레이드

이 접근 방식은 빠른 반복, 낮은 개발 비용을 가능하게 하며, 많은 일반적인 애플리케이션에 충분한 경우가 많습니다. 이 접근 방식은 다음에 가장 적합합니다:
- 초기 프로토타이핑 및 개발
- 엄격한 지연 시간 요구사항이 있는 애플리케이션
- 비용에 민감한 구현
- 대량의 간단한 작업

### 옵션 2: 가장 기능이 풍부한 모델로 시작

지능과 고급 기능이 가장 중요한 복잡한 작업의 경우, 가장 기능이 풍부한 모델로 시작한 다음 나중에 더 효율적인 모델로 최적화하는 것을 고려할 수 있습니다:

1. Claude Sonnet 4.5로 구현
2. 이러한 모델에 대한 프롬프트 최적화
3. 성능이 요구사항을 충족하는지 평가
4. 더 큰 워크플로우 최적화를 통해 시간이 지남에 따라 지능을 낮춰 효율성 증대 고려

이 접근 방식은 다음에 가장 적합합니다:
- 복잡한 추론 작업
- 과학 또는 수학 애플리케이션
- 미묘한 이해가 필요한 작업
- 비용보다 정확성이 중요한 애플리케이션
- 고급 코딩

| 필요한 경우... | 시작하기를 권장합니다... | 사용 사례 예시 |
|------------------|-------------------|-------------------|
| 복잡한 에이전트 및 코딩에 최적의 모델, 대부분의 작업에서 최고의 지능, 장시간 실행되는 자율 작업을 위한 우수한 도구 오케스트레이션 | Claude Sonnet 4.5 | 자율 코딩 에이전트, 사이버 보안 자동화, 복잡한 금융 분석, 수시간 소요되는 연구 작업, 다중 에이전트 프레임워크 |

| 특화된 복잡한 작업을 위한 뛰어난 지능과 추론 | Claude Opus 4.1 | 매우 복잡한 코드베이스 리팩토링, 미묘한 창작 글쓰기, 특화된 과학 분석 |
| 번개 같은 속도와 확장된 사고를 갖춘 거의 최고 수준의 성능 - 가장 경제적인 가격대에서 가장 빠르고 지능적인 Haiku 모델 | Claude Haiku 4.5 | 실시간 애플리케이션, 대량의 지능형 처리, 강력한 추론이 필요한 비용에 민감한 배포, 하위 에이전트 작업 |

## 모델 업그레이드 또는 변경 여부 결정

모델을 업그레이드하거나 변경해야 하는지 결정하려면 다음을 수행해야 합니다:
1. [벤치마크 테스트 생성](/docs/ko/test-and-evaluate/develop-tests) - 사용 사례에 특화된 테스트를 만들어야 합니다. 좋은 평가 세트를 보유하는 것이 프로세스에서 가장 중요한 단계입니다
2. 실제 프롬프트 및 데이터로 테스트
3. 다음 항목에 대해 모델 간 성능 비교:
   - 응답의 정확성
   - 응답 품질
   - 엣지 케이스 처리
4. 성능과 비용 트레이드오프 검토
