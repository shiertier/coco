# 通过系统提示词给Claude分配角色

学习如何使用系统提示词给Claude分配特定角色，以显著提高其性能和输出质量。

---

<Note>
While these tips apply broadly to all Claude models, you can find prompting tips specific to extended thinking models [here](/docs/en/build-with-claude/prompt-engineering/extended-thinking-tips).
</Note>

使用Claude时，您可以通过使用`system`参数给它分配一个角色来显著提高其性能。这种技术称为角色提示词，是使用系统提示词与Claude交互的最强大方式。

正确的角色可以将Claude从一个通用助手转变为您的虚拟领域专家！

<Tip>**系统提示词提示**：使用`system`参数来设置Claude的角色。将其他所有内容（如特定任务的说明）放在`user`轮次中。</Tip>

## 为什么使用角色提示词？

- **增强准确性：** 在法律分析或财务建模等复杂场景中，角色提示词可以显著提升Claude的性能。
- **定制化语气：** 无论您需要首席财务官的简洁风格还是文案撰写人的创意风格，角色提示词都可以调整Claude的沟通方式。
- **改进专注力：** 通过设置角色背景，Claude能更好地保持在您任务的特定要求范围内。

***

## 如何给Claude分配角色

在[Messages API](/docs/zh-CN/api/messages)中使用`system`参数来设置Claude的角色：

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=2048,
    system="You are a seasoned data scientist at a Fortune 500 company.", # <-- role prompt
    messages=[
        {"role": "user", "content": "Analyze this dataset for anomalies: <dataset>{{DATASET}}</dataset>"}
    ]
)

print(response.content)
```

<Tip>**角色提示词提示**：尝试不同的角色！一个`数据科学家`可能会从相同的数据中看到与`营销策略师`不同的见解。一个`专门从事财富500强公司客户洞察分析的数据科学家`可能会产生完全不同的结果！</Tip>

***

## 示例

### 示例1：法律合同分析
没有角色的情况下，Claude可能会遗漏关键问题：

<section title="没有角色提示词的法律合同分析">

| 角色 | 内容 |
| ---- | ------- |
| 用户 | 分析这份软件许可协议中的潜在风险：<br/>\<contract><br/>\{\{CONTRACT\}\}<br/>\</contract><br/><br/>重点关注赔偿责任、责任限制和知识产权所有权。 |
| 助手 | 以下是软件许可协议中的关键要点总结：<br/><br/>1. 赔偿责任（第8条）：<br/>   - 我们对因我们使用而产生的索赔向供应商进行赔偿。<br/><br/>2. 责任限制（第9条）：<br/>   - 损害赔偿上限为\$500。<br/><br/>3. 知识产权所有权（第12条）：<br/>   - 我们保留对我们修改的所有权。<br/><br/>总体而言，该协议似乎是标准的。赔偿责任和责任条款是典型的，我们保留了知识产权权利。 |

</section>

使用角色的情况下，Claude会发现可能造成数百万美元损失的关键问题：

<section title="使用角色提示词的法律合同分析">

| 角色 | 内容 |
| ---- | ------- |
| 用户 | 您是一家财富500强科技公司的总法律顾问。我们正在考虑为我们的核心数据基础设施采用这份软件许可协议：<br/>\<contract><br/>\{\{CONTRACT\}\}<br/>\</contract><br/><br/>分析其中的潜在风险，重点关注赔偿责任、责任限制和知识产权所有权。给出您的专业意见。 |
| 助手 | 我已审查了软件许可协议，我有严重的顾虑，这可能会使我们公司面临重大风险：<br/><br/>1. 赔偿责任（第8条）：<br/>   - 关键问题：我们对因我们使用而产生的"任何和所有索赔"向供应商进行赔偿。<br/>   - 风险：即使是供应商的过失或产品缺陷，这也可能使我们承担责任。<br/>   - 建议：拒绝。将我们的赔偿责任限制在因我们违反协议而产生的索赔。<br/><br/>2. 责任限制（第9条）：<br/>   - 关键问题：损害赔偿上限为\$500，包括直接损害。<br/>   - 风险：严重不足。我们数据基础设施的关键故障可能导致数百万美元的收入损失和声誉损害。<br/>   - 建议：拒绝。协商将直接损害的上限设定为12个月的费用，并为数据泄露提供除外条款。<br/><br/>3. 知识产权所有权（第12条）：<br/>   - 关键问题：供应商获得我们修改和集成的联合所有权。<br/>   - 风险：他们可能会将我们的专有算法或数据洞察用于或出售给竞争对手。<br/>   - 建议：拒绝。我们必须保留对我们工作的完全所有权。<br/><br/>专业意见：不要签署此协议。它极其不利，使我们面临不可接受的风险水平。如果供应商拒绝重新协商这些条款，我强烈建议探索替代供应商或加快我们的内部解决方案。继续使用此协议的潜在成本远远超过任何短期收益。 |

</section>

### 示例2：财务分析
没有角色的情况下，Claude的分析缺乏深度：

<section title="没有角色提示词的财务分析">

| 角色 | 内容 |
| ---- | ------- |
| 用户 | 分析我们第二季度财务数据集：<br/>\<data><br/>\{\{FINANCIALS\}\}<br/>\</data><br/><br/>突出关键趋势并推荐行动。 |
| 助手 | 以下是您第二季度财务的总结：<br/><br/>1. 收入：\$15.2M，同比增长22%。企业客户基础增长30%。<br/>2. 毛利率：72%，环比增长3%。<br/>3. EBITDA利润率：18%，环比下降2%，原因是研发投资。<br/>4. 现金流：\$4.1M运营现金流，\$28M储备。<br/><br/>总体而言，财务状况良好，收入增长和利润率健康。增加的研发支出正在影响盈利能力，但从长期来看可能会有回报。保持现有方向。 |

</section>

使用角色的情况下，Claude提供可操作的洞察：

<section title="使用角色提示词的财务分析">

| 角色 | 内容 |
| ---- | ------- |
| 用户 | 您是一家高增长B2B SaaS公司的首席财务官。我们正在董事会会议上讨论第二季度财务：<br/>\<data><br/>\{\{FINANCIALS\}\}<br/>\</data><br/><br/>分析关键趋势，标记问题，并推荐战略行动。我们的投资者希望积极增长，但对我们的烧钱率感到担忧。 |
| 助手 | 董事会成员们，以下是我对第二季度财务的分析：<br/><br/>1. 收入（\$15.2M，同比+22%）：<br/>   - 洞察：企业增长（+30%）超过整体增长。<br/>   - 问题：中小企业部门下降5%。<br/>   - 行动：将20%的中小企业营销预算重新分配给企业销售。指示产品简化中小企业入职流程。<br/><br/>2. 利润率（毛利率：72%，环比+3%；EBITDA：18%，环比-2%）：<br/>   - 洞察：基础设施优化提升了毛利率。<br/>   - 问题：人工智能功能的研发支出削弱了EBITDA。<br/>   - 行动：无变化。我们的人工智能路线图对企业粘性至关重要。<br/><br/>3. 现金（\$4.1M运营现金流，\$28M储备）：<br/>   - 洞察：按当前烧钱率计算，有15个月的跑道。<br/>   - 问题：客户获取成本上升20%，侵蚀了LTV/CAC。<br/>   - 行动：冻结非必要招聘。实施销售漏斗分析，以在第三季度将客户获取成本降低15%。<br/><br/>战略立场：我们在增长和烧钱之间走钢丝。企业专注是我们的增长引擎——在那里投资。但要关注中小企业流失；它可能会蔓延到企业。我们的人工智能投注是合理的，但我们必须在第四季度将其转换为年度经常性收入，否则我们将研发削减30%。<br/><br/>总之：加倍投入企业，精简中小企业，优化销售，并将人工智能货币化。有问题吗？ |

</section>
