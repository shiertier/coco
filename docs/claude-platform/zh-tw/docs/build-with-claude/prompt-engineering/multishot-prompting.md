# 使用範例（多次提示）來引導 Claude 的行為

通過在提示中提供精心設計的範例，可以顯著改進 Claude 輸出的準確性、一致性和品質。

---

<Note>
While these tips apply broadly to all Claude models, you can find prompting tips specific to extended thinking models [here](/docs/en/build-with-claude/prompt-engineering/extended-thinking-tips).
</Note>

範例是讓 Claude 生成您所需內容的秘密武器捷徑。通過在提示中提供幾個精心設計的範例，您可以顯著改進 Claude 輸出的準確性、一致性和品質。
這種技術被稱為少量示例或多次提示，特別適用於需要結構化輸出或遵守特定格式的任務。

<Tip>**增強您的提示**：包含 3-5 個多樣化、相關的範例來向 Claude 展示您確切想要的內容。更多範例 = 更好的性能，特別是對於複雜任務。</Tip>

## 為什麼使用範例？

- **準確性**：範例減少對指令的誤解。
- **一致性**：範例強制統一的結構和風格。
- **性能**：精心選擇的範例提升 Claude 處理複雜任務的能力。

## 製作有效的範例

為了最大效果，請確保您的範例具有：
- **相關性**：您的範例反映您的實際使用案例。
- **多樣性**：您的範例涵蓋邊界情況和潛在挑戰，並且變化足夠大，使 Claude 不會無意中拾取意外的模式。
- **清晰性**：您的範例用 `<example>` 標籤包裝（如果有多個，嵌套在 `<examples>` 標籤內）以便結構化。

<Tip>要求 Claude 評估您的範例的相關性、多樣性或清晰性。或者讓 Claude 根據您的初始集合生成更多範例。</Tip>

    | 助手（Claude 的回應） | 以下是客戶反饋的分析：<br/><br/>問題：UI/UX<br/>情感：負面<br/>優先級：高<br/>客戶對新儀表板設計感到沮喪，他們認為設計混亂且加載速度緩慢。他們特別指出缺少「導出」按鈕，這可能是一個關鍵功能。此問題需要作為高優先級進行處理。<br/><br/>問題：集成<br/>情感：正面<br/>優先級：中<br/>客戶對現有的 Salesforce 集成感到滿意，但希望看到對 Hubspot 的支持。這是一個功能請求，可以考慮用於未來的產品更新。<br/><br/>問題：定價<br/>情感：負面<br/>優先級：高<br/>客戶認為定價不合理，特別是對於基本功能。他們威脅要轉向競爭對手，因此這是一個需要審查的高優先級問題。<br/><br/>問題：性能<br/>情感：負面<br/>優先級：高<br/>客戶在移動應用上遇到關鍵問題，大文件上傳始終導致崩潰。這是一個重大問題，影響客戶有效使用產品的能力，應作為高優先級進行處理。<br/><br/>問題：UI/UX<br/>情感：正面<br/>優先級：低<br/>客戶對產品總體滿意，特別指出直觀的 UI 和優秀的客戶支持。這是一個正面反饋，不需要立即採取行動。 | 以下是客戶反饋的分析：<br/><br/>1. 新儀表板亂七八糟！加載速度很慢，我找不到導出按鈕。立即修復！<br/>類別：UI/UX、性能<br/>情感：負面<br/>優先級：高<br/><br/>2. 喜歡 Salesforce 集成！但如果您能添加 Hubspot 就太好了。<br/>類別：集成、功能請求<br/>情感：正面<br/>優先級：中<br/><br/>3. 您的定價太荒謬了！基本功能每月 $99？我要轉向競爭對手 X。<br/>類別：定價<br/>情感：負面<br/>優先級：高<br/><br/>4. 當我嘗試上傳大文件（>50MB）時，移動應用不斷崩潰。這對我們的現場代理至關重要。<br/>類別：性能<br/>情感：負面<br/>優先級：高<br/><br/>5. 總的來說，我對產品很滿意。UI 直觀，客戶支持一流。繼續加油！<br/>類別：UI/UX、其他（客戶支持）<br/>情感：正面<br/>優先級：低 |
