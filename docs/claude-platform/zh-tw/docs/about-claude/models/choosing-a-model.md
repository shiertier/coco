# 選擇正確的模型

選擇適合您應用程式的最佳 Claude 模型涉及平衡三個關鍵考量：功能、速度和成本。本指南可幫助您根據特定需求做出明智的決定。

---

## 建立關鍵標準

選擇 Claude 模型時，我們建議您先評估以下因素：
- **功能：** 模型需要具備哪些特定功能或能力才能滿足您的需求？
- **速度：** 模型在您的應用程式中需要多快地做出回應？
- **成本：** 您在開發和生產使用方面的預算是多少？

提前知道這些答案將使縮小範圍和決定使用哪個模型變得容易得多。

***

## 選擇最佳的起始模型

有兩種一般方法可用於開始測試哪個 Claude 模型最適合您的需求。

### 選項 1：從快速、經濟高效的模型開始

對於許多應用程式，從更快、更經濟高效的模型（如 Claude Haiku 4.5）開始可能是最佳方法：

1. 使用 Claude Haiku 4.5 開始實施
2. 徹底測試您的使用案例
3. 評估效能是否符合您的要求
4. 僅在特定功能差距需要時才升級

這種方法允許快速迭代、較低的開發成本，通常對許多常見應用程式都足夠。這種方法最適合：
- 初始原型設計和開發
- 具有嚴格延遲要求的應用程式
- 成本敏感的實施
- 高容量、直接的任務

### 選項 2：從最強大的模型開始

對於複雜任務，其中智能和先進功能至關重要，您可能希望從最強大的模型開始，然後考慮隨著時間的推移優化到更高效的模型：

1. 使用 Claude Sonnet 4.5 實施
2. 為這些模型優化您的提示
3. 評估效能是否符合您的要求
4. 考慮通過更大的工作流程優化隨著時間推移降低智能來提高效率

這種方法最適合：
- 複雜推理任務
- 科學或數學應用程式
- 需要細緻理解的任務
- 準確性優於成本考量的應用程式
- 進階編碼

## 決定是否升級或更改模型

要確定是否需要升級或更改模型，您應該：
1. [建立基準測試](/docs/zh-TW/test-and-evaluate/develop-tests)特定於您的使用案例 - 擁有良好的評估集是流程中最重要的步驟
2. 使用您的實際提示和資料進行測試
3. 比較模型之間的效能：
   - 回應的準確性
   - 回應品質
   - 邊界情況的處理
4. 權衡效能和成本的權衡
