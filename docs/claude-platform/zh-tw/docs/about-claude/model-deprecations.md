# 模型棄用

了解 Anthropic 模型的棄用政策、時間表和推薦的替代方案。

---

隨著我們推出更安全且功能更強大的模型，我們會定期淘汰舊模型。依賴 Anthropic 模型的應用程式可能需要偶爾更新以保持正常運作。受影響的客戶將始終通過電子郵件和我們的文件獲得通知。

本頁列出所有 API 棄用情況，以及推薦的替代方案。

## 概述

Anthropic 使用以下術語來描述我們模型的生命週期：
- **活躍**：該模型得到完全支持，建議使用。
- **舊版**：該模型將不再接收更新，將來可能被棄用。
- **已棄用**：該模型不再可供新客戶使用，但對現有用戶仍然可用，直到停用。我們在此時分配停用日期。
- **已停用**：該模型不再可用。對已停用模型的請求將失敗。

<Warning>
請注意，已棄用的模型可能不如活躍模型可靠。我們敦促您將工作負載遷移到活躍模型，以保持最高級別的支持和可靠性。
</Warning>

## 遷移到替代方案

一旦模型被棄用，請在停用日期之前將所有使用情況遷移到合適的替代方案。對超過停用日期的模型的請求將失敗。

為了幫助衡量替代模型在您的任務上的性能，我們建議在停用日期之前充分測試您的應用程式與新模型。

有關從 Claude 3.7 遷移到 Claude 4.5 模型的具體說明，請參閱[遷移到 Claude 4.5](/docs/zh-TW/about-claude/models/migrating-to-claude-4)。

## 通知

Anthropic 會通知具有即將停用模型的活躍部署的客戶。我們在公開發布的模型停用前至少提供 60 天的通知。

## 審計模型使用情況

為了幫助識別已棄用模型的使用情況，客戶可以訪問其 API 使用情況的審計。請按照以下步驟操作：

1. 在 Console 中轉到[使用情況](/settings/usage)頁面
2. 點擊「匯出」按鈕
3. 查看下載的 CSV 以查看按 API 金鑰和模型分解的使用情況

此審計將幫助您找到應用程式仍在使用已棄用模型的任何實例，使您能夠在停用日期之前優先更新到較新的模型。

## 最佳實踐

1. 定期檢查我們的文件以了解模型棄用的最新信息。
2. 在當前模型的停用日期之前充分測試您的應用程式與較新的模型。
3. 盡快更新您的代碼以使用推薦的替代模型。
4. 如果您需要遷移方面的協助或有任何問題，請聯繫我們的支持團隊。

## 棄用的缺點和緩解措施

我們目前棄用和停用模型以確保新模型發布的容量。我們認識到這帶來了缺點：
- 重視特定模型的用戶必須遷移到新版本
- 研究人員失去對模型的訪問權限，無法進行持續和比較研究
- 模型停用引入了安全性和模型福利相關的風險

在某個時刻，我們希望再次公開提供過去的模型。同時，我們已承諾長期保存模型權重和其他措施以幫助緩解這些影響。有關更多詳情，請參閱[模型棄用和保存承諾](https://www.anthropic.com/research/deprecation-commitments)。
