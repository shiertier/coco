智能体是 OpenAI Agents SDK 的主要构建块。一个**Agent**（智能体）是一个已配置的 Large Language Model（LLM），包含：

- **Instructions** —— 指示模型“它是谁”和“它应如何回应”的系统提示。
- **Model** —— 要调用的 OpenAI 模型，以及可选的模型调优参数。
- **Tools** —— LLM 可调用以完成任务的一组函数或 API。

## 基本配置

`Agent` 构造函数接收单个配置对象。最常用的属性如下所示。

| Property        | Required | Description                                                                              |
| --------------- | -------- | ---------------------------------------------------------------------------------------- |
| `name`          | yes      | 简短且可读的标识符。                                                                     |
| `instructions`  | yes      | 系统提示（字符串**或**函数——见[动态 instructions](#dynamic-instructions)）。             |
| `model`         | no       | 模型名称**或**自定义 [`Model`](/openai-agents-js/openai/agents/interfaces/model/) 实现。 |
| `modelSettings` | no       | 调优参数（temperature、top_p 等）。如果所需属性不在顶层，可放在 `providerData` 下。      |
| `tools`         | no       | 模型可调用的 [`Tool`](/openai-agents-js/openai/agents/type-aliases/tool/) 实例数组。     |

## 上下文

智能体对其上下文类型是**泛型**的——即 `Agent<TContext, TOutput>`。该上下文是一个依赖注入对象，由你创建并传给 `Runner.run()`。它会被转发到每个工具、护栏、交接等，适合用于存储状态或提供共享服务（数据库连接、用户元数据、功能开关等）。

## 输出类型

默认情况下，智能体返回**纯文本**（`string`）。如果你希望模型返回结构化对象，可以指定 `outputType` 属性。SDK 接受：

1. 一个 [Zod](https://github.com/colinhacks/zod) schema（`z.object({...})`）。
2. 任意兼容 JSON Schema 的对象。

## 多智能体系统设计模式

有很多方式可以将智能体组合在一起。我们在生产应用中常见的两种模式是：

1. **管理器（智能体作为工具）**——一个中心智能体拥有对话，并调用以工具形式暴露的专用智能体。
2. **交接**——初始智能体在识别出用户请求后，将整个对话交给专家智能体。

这些方法是互补的。管理器让你可以在一个地方实施护栏或速率限制，而交接则让每个智能体专注于单一任务而无需持续控制对话。

### 管理器（智能体作为工具）

在此模式中，管理器从不移交控制权——LLM 使用工具，且由管理器总结最终答案。详见[工具](/openai-agents-js/zh/guides/tools#agents-as-tools)。

<Code lang="typescript" code={agentsAsTools} title="Agents as tools" />

### 交接

通过交接，分诊智能体负责路由请求，但一旦交接发生，专家智能体就拥有对话，直至产出最终结果。这样可以让提示更简洁，并使你能独立地推理每个智能体。了解更多见[交接](/openai-agents-js/zh/guides/handoffs)。

## 强制使用工具

提供工具并不保证 LLM 一定会调用。你可以通过 `modelSettings.tool_choice` **强制**使用工具：

1. `'auto'`（默认）——LLM 决定是否使用工具。
2. `'required'`——LLM 必须调用某个工具（可自行选择）。
3. `'none'`——LLM 必须**不**调用工具。
4. 指定工具名，例如 `'calculator'`——LLM 必须调用该特定工具。

<Code lang="typescript" code={agentForcingToolUse} title="Forcing tool use" />

### 防止无限循环

在一次工具调用后，SDK 会自动将 `tool_choice` 重置回 `'auto'`。这可防止模型陷入反复尝试调用工具的无限循环。你可以通过 `resetToolChoice` 标志或配置 `toolUseBehavior` 覆盖此行为：

- `'run_llm_again'`（默认）——使用工具结果再次运行 LLM。
- `'stop_on_first_tool'`——将第一次工具结果视为最终答案。
- `{ stopAtToolNames: ['my_tool'] }`——当任一列出的工具被调用时停止。
- `(context, toolResults) => ...`——返回是否应结束本次运行的自定义函数。

```typescript
const agent = new Agent({
  ...,
  toolUseBehavior: 'stop_on_first_tool',
});
```
